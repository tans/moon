# MOON 产品设计

## 1. 产品定位

**产品名**：MOON  
**释义**：Model Always Online

MOON 是一个多模型聚合产品。

核心逻辑：
- 优先使用高层模型
- 高层额度用完后切换到中层模型
- 中层额度用完后进入基础模型
- 基础模型始终在线

核心表达：
- 大模型一直在线
- 从满月到新月，始终可用

---

## 2. 产品目标

MOON 解决的问题：
- 用户不想自己维护多个模型账号
- 用户不想理解不同模型接口和计费方式
- 用户需要一个统一入口使用不同模型
- 用户希望服务始终可用，不因为高层额度耗尽而中断

产品目标：
- 用统一入口聚合多个模型
- 用月亮状态表达模型层级
- 用订阅套餐完成商业化
- 用简单规则降低理解成本

---

## 3. 产品结构

MOON 由三部分组成：

### 3.1 模型层级
- 满月
- 半月
- 新月

### 3.2 商业套餐
- 入门
- 普通
- 高级

### 3.3 核心界面
- 首页
- 对话页
- 配置页
- 订阅页

模型层级用于表达系统路由，不直接作为套餐名。
套餐名用于销售和计费。

---

## 4. 模型层级设计

### 满月
定位：最高层模型

建议模型：
- GPT
- Gemini
- Claude

适合任务：
- 复杂任务
- 编程
- 长文本
- 复杂推理

规则：
- 默认优先使用满月模型
- 满月额度按日计算

### 半月
定位：中间层模型

建议模型：
- Kimi
- MiniMax
- Qwen

适合任务：
- 日常任务
- 总结
- 改写
- 翻译
- 常规问答

规则：
- 满月额度不足后切换到半月模型
- 半月额度按日计算

### 新月
定位：基础层模型

建议模型：
- 快速模型
- 低成本模型
- 基础对话模型

适合任务：
- 聊天
- 续写
- 润色
- 简单生成

规则：
- 新月不限量
- 始终在线

## 5. 套餐设计

### 入门
价格：**¥9.9 / 月**

权益：
- 满月 30 次 / 天
- 半月 200 次 / 天
- 新月不限
- 自动路由

适合人群：
- 轻度用户
- 尝鲜用户
- 基础办公用户

### 普通
价格：**¥39 / 月**

权益：
- 满月 200 次 / 天
- 半月 1000 次 / 天
- 新月不限
- 更稳定的优先队列

适合人群：
- 日常主力用户
- 写作用户
- 学习和办公用户

### 高级
价格：**¥99 / 月**

权益：
- 满月 1000 次 / 天
- 半月 5000 次 / 天
- 新月不限
- 长上下文
- 高优先级

适合人群：
- 高频用户
- 重度创作用户
- 团队试用用户

## 6. 首页设计

首页目标：
- 用一句话说明产品价值
- 用月亮状态说明模型路由逻辑
- 用套餐说明完成转化

当前首页结构：

### 6.1 顶部导航
- Logo：MOON
- 副标题：Model Always Online
- 按钮：开始体验

### 6.2 Hero 区
- 主标题：大模型一直在线
- 副说明：优先使用满月模型，其次半月模型，新月不限
- CTA：立即开始 / 查看套餐

### 6.3 状态说明区
- 满月：复杂任务、长文本、编程
- 半月：日常任务、总结、改写、翻译
- 新月：新月不限，聊天、改写、总结

### 6.4 状态模型表
- 满月 → GPT / Gemini / Claude
- 半月 → Kimi / MiniMax / Qwen
- 新月 → 快速模型 / 低成本模型

### 6.5 套餐区
- 入门
- 普通
- 高级

## 8. 交互设计

### 8.1 路由逻辑
默认自动路由：
1. 优先满月
2. 满月不足后使用半月
3. 半月不足后进入新月
4. 新月始终可用

### 8.2 用户感知
用户不需要理解复杂模型参数，只看到：
- 当前层级
- 今日次数
- 当前可用状态

### 8.3 文案原则
- 不用情绪化文案
- 不用修饰性形容词
- 只保留信息表达

文案格式建议：
- 满月优先
- 半月其次
- 新月不限
- 复杂任务 / 日常任务 / 聊天总结
